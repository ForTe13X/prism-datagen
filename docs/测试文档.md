## 1. 测试策略与不变量（Invariants）

本项目的测试不是零散地覆盖函数，而是围绕**几条支柱性不变量**组织。每个不变量对应生成器最核心的
承诺，测试文件基本按不变量分工。这些不变量直接对应生成器的设计契约：真值先建、两个旋钮只动观测、
判别力有区间、物化正确。

| # | 不变量 | 含义 | 主要守护文件 |
|---|--------|------|--------------|
| I1 | **确定性 / 逐字节可复现** | 相同 `(seed, dirtiness, link_explicitness, domain)` → 逐字节相同的包；无 `random`、无时钟（唯一熵源是 `sha256`→`[0,1]`，见 `datagen/core.py` 的 `_unit`/`_wiggle`）。 | `test_determinism.py` |
| I2 | **真值先建、始终可恢复** | `ground_truth` 在任何观测被写出之前就已确定；oracle 恒得满分；答案按**观测可见的 id**（新闻 id / 仓库 id）编键，使只看观测 store 的解法也能被打分。 | `test_truth_recovery.py` |
| I3 | **旋钮只动观测、永不动真值** | `dirtiness` 与 `link_explicitness` 在 `(d, l)` 全网格下都**不改变** `ground_truth`；污染只在观测层发生，并逐条记入可逆的 `corruption_map`（variant→canonical）。 | `test_determinism.py` / `test_dirtiness.py` |
| I4 | **判别区间（discriminative interval）** | link 显式（L1）时连字面 `naive` 也满分；link 隐晦（L4）时 `naive` 崩、`linked`（跨源联结）仍能部分恢复 ⇒ 任务确实需要跨源推理。脏度升高时 `linked` 在**均值上**退化（鲁棒性曲线）。 | `test_solvers.py` |
| I5 | **物化正确** | JSON 可回读（round-trip）、每 store 一个 CSV 行数与包一致、SQLite 是**真实可查询**的关系库、preview 为确定性文本。 | `test_materialize.py` |
| I6 | **CLI 正确接线** | `list` / `gen` / `sweep` 子命令退出码正确、`gen --out` 真的写出文件、未知数据域返回非零退出码。 | `test_cli.py` |

设计上值得注意的一点：**测试全部通过公共 API（`from datagen import ...`）驱动**，不 monkeypatch 内部；
因此这些测试同时也是生成器对外契约的可执行文档。

### 2.1 test_determinism.py（5 用例）

守护不变量 **I1（确定性）** 与 **I3（真值不被旋钮改动）**。用 `_canon(pkg)`
（`json.dumps(..., sort_keys=True)`）把整个包规范化为字符串再逐字节比较。

| 用例 | 断言什么 |
|------|----------|
| `test_same_knobs_byte_identical` | 相同 `(dirtiness=0.6, link=4, seed="ho-0")` 连生两次 → 规范化后**完全相等**。这是最关键的载重属性：可复现。 |
| `test_seed_changes_output_but_stays_reproducible` | 同一 seed（`ho-0`）两次生成相等；不同 seed（`ho-0` vs `ho-1`）生成**不相等**。即 seed 真是唯一熵源且确实生效。 |
| `test_truth_invariant_to_dirtiness_and_link` | 在 `dirtiness ∈ {0.0,0.3,0.6,0.9}` × `link ∈ {1..5}` 的**全 20 格网格**上，`ground_truth.answers` 恒等于基准（`d=0,l=1`）的答案 —— 两个面向观测的旋钮**永不**改变真值答案。 |
| `test_knobs_are_clamped` | 越界输入被夹紧：`dirtiness=-5→0.0`、`link=-9→1`；`dirtiness=99→1.0`、`link=99→5`。 |
| `test_unknown_domain_returns_none` | `generate("does_not_exist")` 返回 `None`（未知域不抛异常）。 |

### 2.2 test_truth_recovery.py（5 用例）

守护不变量 **I2（真值先建、始终可恢复）**。

| 用例 | 断言什么 |
|------|----------|
| `test_oracle_is_perfect` | 在 `dirtiness ∈ {0.0,0.6,0.9}` 下，`oracle_solve` 对 `explain_delays` 的 `score.f1 == 1.0` —— 知道真值的解法在**任何脏度**下都满分，证明真值确实可无损恢复。 |
| `test_events_on_distinct_hubs` | 所有 ground-truth 事件落在**互不相同**的 hub（`record_hub_fk` 去重后数量不变）。这是「按 hub 编键的异常/答案映射不会把两条真值折叠成一条」的基石。 |
| `test_answers_keyed_by_observable_news_ids` | `explain_delays` 的答案键是**新闻 store 里真实存在的 news id**（`set(ans) ⊆ news_ids` 且非空）—— 只看观测 store 的解法可被打分。 |
| `test_observation_view_hides_truth_metadata` | `observation_view(pkg)` 里每条新闻都**不含** `_truth_event` 内部标签，且整体**不含** `ground_truth` —— 真值绝不泄漏进观测通道。 |
| `test_affected_shipments_match_events` | 每个事件声称延误的每一张运单，其 `record_hub_fk` 确实等于该事件的 hub —— 真值内部自洽，不会张冠李戴。 |

### 2.3 test_dirtiness.py（5 用例）

守护不变量 **I3（污染只动观测、可逆、随 d 缩放）**。对应 `_apply_dirtiness` 的 6 类污染
（别名 / 单位漂移 kg→lb / 状态置空 / 时间偏移 / 数值冻结 / 编码乱码）。

| 用例 | 断言什么 |
|------|----------|
| `test_zero_dirtiness_is_clean` | `dirtiness=0.0` 时 `corruption_map` 各类均空；所有运单 `unit=="kg"`、`status` 非 `None`。零脏度即绝对干净。 |
| `test_corruption_scales_with_dirtiness` | 污染总条数：`total(0.0)==0` 且 `total(0.9) > total(0.3) > 0` —— 脏度越高、污染越多。 |
| `test_unit_drift_recorded_and_reversible` | 在 `d=0.9` 下，运单**恰好**是被 `weight_lb_ids` 记录的那些以 `lb` 存储（`(unit=="lb") == (id in lb_ids)`）—— 打分方可据此换算回 kg，污染完全可逆。 |
| `test_garbled_news_flagged` | `garbled_news` 里被标乱码的每个 id 都真实存在于新闻 store（`corruption_map` 是污染的忠实索引）。 |
| `test_dirtiness_never_touches_ground_truth` | 同一 seed 下 `dirtiness=0.0` 与 `0.9` 生成的 `ground_truth` **完全相等** —— 脏度绝不碰真值。 |

### 2.4 test_solvers.py（5 用例）

守护不变量 **I4（判别区间与鲁棒性曲线）**。三个解法：`oracle`（知真值，恒 1.0）/
`naive`（字面精确匹配、单源）/ `linked`（跨源时空 + 实体联结）。

| 用例 | 断言什么 |
|------|----------|
| `test_naive_perfect_when_link_is_literal` | `link=1`（新闻字面写出运单 id）时 `naive_f1 == 1.0` —— link 显式则字面解法即可满分。 |
| `test_naive_collapses_when_link_hidden` | `link=4`（时空散文、无字面 id/hub）时 `naive_f1 < linked_f1` 且 `gap > 0` —— link 隐晦处 naive 崩、linked 胜。 |
| `test_linked_beats_naive_in_discriminative_interval` | 在 `link=4` 上跨 8 个 seed（`ho-0..ho-7`）的 **gap 均值 > 0** —— 跨源任务确有「区分度」。 |
| `test_dirtiness_degrades_linked_on_the_mean` | `linked` 的 8-seed 均值满足 `mean(0.9) < mean(0.0)` 且 `mean(0.0) ≥ mean(0.3) ≥ mean(0.6) ≥ mean(0.9)` —— **均值上**单调的鲁棒性曲线（注意：是均值，非逐 seed，详见 [第 4 节](#4-一段真实的测试演进诚实纪律的例子)）。 |
| `test_oracle_always_one` | `evaluate(...)["oracle_f1"] == 1.0`。 |

> 诚实边界（务必知晓）：确定性 `linked` 解法只区分 **L1（显式 key）vs L≥2（非显式）**；
> L2–L5 得分相同（约 0.8）。它 OR 了多个线索（名称/区域/港口/异常时刻），每个非显式级别都至少留下
> 一个线索，故无法拉开 L2→L5 的细梯度。它是**未来 LLM/axiom 语义解法的确定性占位替身**，
> 不是真正的语义求解器；`linked` 的满分上限也是约 **0.8** 而非 1.0（骨架解法上限）。
> 上表的用例断言的是这些**结构性关系**（naive 崩、linked 相对更高、均值单调），并未断言 L2≠L3≠L4≠L5。

### 2.5 test_materialize.py（4 用例）

守护不变量 **I5（物化正确）**。覆盖 `to_json` / `to_csv` / `to_sqlite` / `preview` 四种落地形态。

| 用例 | 断言什么 |
|------|----------|
| `test_json_roundtrips` | `to_json` 写出后回读，`ground_truth.answers` 与 `manifest.counts` 与原包一致（round-trip 无损）。 |
| `test_csv_row_counts` | `to_csv` 写出的文件名集合含 `warehouses/carriers/shipments/throughput/news.csv`；且 `shipments.csv` 行数等于包内 shipments 数（用 `utf-8-sig` 读，兼容 Excel BOM）。 |
| `test_sqlite_is_queryable` | `to_sqlite` 产出**真实 SQLite**：执行 `SELECT COUNT(*) FROM shipments WHERE status='delayed'`，结果等于包内 `status=="delayed"` 的运单数 —— 关系库能回答与包一致的真实查询。 |
| `test_preview_is_deterministic_text` | `preview(pkg)` 两次调用**逐字符相等**（无时钟），且文本含关键锚点 `"跨源真值"` 与 `"corruption_map"`。 |

### 2.6 test_cli.py（5 用例）

守护不变量 **I6（CLI 正确接线）**。直接调用 `datagen.cli.main([...])` 检查退出码与 stdout（用 pytest 的
`capsys` / `tmp_path` fixture）。

| 用例 | 断言什么 |
|------|----------|
| `test_list_ok` | `main(["list"]) == 0` 且输出含 `logistics_demo`。 |

## 3. 如何运行测试

前置：Python 3.10+（源码用了 `X | None` 语法与 `from __future__ import annotations`）与 `pytest`。
**除 `pytest` 外无第三方依赖**——被测代码仅用标准库（`hashlib`/`json`/`sqlite3`/`csv`/`argparse`/`math`/`pathlib`）。

在项目根目录（`prism-datagen/`）执行：

```bash
python -m pytest tests/ -q
```

实测输出：

```
.............................                                            [100%]
29 passed in 0.28s
```

常用变体：

```bash
python -m pytest tests/ -v                       # 逐用例列名
python -m pytest tests/test_solvers.py -q        # 只跑判别力那组
python -m pytest tests/ --collect-only -q        # 只收集不执行(核对 29 个用例)
```

全套 29 个用例在毫秒到亚秒级完成（无网络、无外部进程、无大数据），可放心作为提交前门禁。

## 4. 一段真实的测试演进（诚实纪律的例子）

`test_dirtiness_degrades_linked_on_the_mean` 这条用例并非一开始就是现在这样。

**最初的写法**断言的是「**逐 seed 单调**下降」——即对固定 seed，脏度越高 `linked_f1` 就应越低。
这个直觉看似合理（linked 依赖区域/港口/时间线索，污染这些线索理应削弱它），但**实测会失败**。

真实的反例（本次运行复现，`link=4`，seed `ho-1`）：

```
ho-1:  clean(d=0.0) linked_f1 = 0.714     dirty(d=0.9) linked_f1 = 0.727   ← 脏 0.9 反而更高
```

原因是：污染并非逐 seed 单调。在**单个 seed** 上，某次污染偶尔会把一个「边界匹配」歪打正着地
翻到正确一侧（例如时间偏移恰好把新闻帧推进 `linked` 的容差窗内），于是脏一点反而分更高。
逐 seed 单调这个断言因此是**过强、且事实上不成立**的。

**修正后的写法**改为断言「**跨 8 seed 均值单调**」。这既是真实成立的趋势，也如实承认了单 seed 的
非单调性。本次运行实测的 8-seed（`ho-0..ho-7`）`linked_f1` 均值：

```
dirtiness   mean linked_f1 (over 8 seeds)
   0.0            0.667
   0.3            0.627
   0.6            0.573
   0.9            0.554
```

`0.667 → 0.627 → 0.573 → 0.554`，均值上确实单调下降。这就是当前用例断言的两条：
`mean(0.9) < mean(0.0)`（重污染在均值上明显有害）与 `mean(0.0) ≥ mean(0.3) ≥ mean(0.6) ≥ mean(0.9)`
（均值单调曲线）。

这段演进是本项目**诚实纪律**的具体样本：当一条断言在真实数据上站不住脚时，**不是**去挑一个「好看」的
seed 把测试凑绿，而是把断言弱化到「真实成立的那个层级（均值趋势）」，并在测试的 docstring 里
白纸黑字写明「NOT strictly per-seed」的理由。测试文件的注释原文即保留了这条说明。

## 5. 判别力与鲁棒性的真实数字

以下均为 `python -m datagen sweep` 的真实输出（默认域 `logistics_demo`，用 spec 默认 seed），可与
测试断言互相印证。

**扫 link（固定 `dirtiness=0.0`）—— 判别区间：**

```
   link | naive_f1  linked_f1   gap
   -----+----------------------------
      1 |   1.000     0.800    -0.200
      2 |   0.000     0.800    +0.800
      3 |   0.000     0.800    +0.800
      4 |   0.000     0.800    +0.800
      5 |   0.000     0.800    +0.800
```

解读：**L1** 新闻字面写出 id，连 `naive` 都满分（`gap` 为负仅因 linked 骨架上限 ~0.8 而非 1.0，
非缺陷）；**L2–L5** naive 全崩到 0、linked 稳在 0.800，`gap=+0.800`。这正是「任务需要跨源推理」的证据。
（如前所述，L2–L5 得分相同是确定性骨架的已知局限，非真实语义梯度。）

**扫 dirtiness（固定 `link=4`）—— 鲁棒性曲线（此处为 spec 默认 seed 的单条曲线）：**

```
 dirtiness | naive_f1  linked_f1   gap
 ----------+----------------------------
    0.0    |   0.000     0.800    +0.800
    0.3    |   0.000     0.571    +0.571
    0.6    |   0.000     0.500    +0.500
    0.9    |   0.000     0.500    +0.500
```

解读：`naive` 在 L4 恒为 0；`linked` 随脏度从 0.800 退化到 0.500。注意这是**默认单 seed** 的曲线，
与第 4 节「8-seed 均值曲线」数值不同（单 seed 会有台阶/回弹），二者一起才是完整、诚实的鲁棒性画面。

另一组真实 CLI 输出（`gen -d 0.6 -l 4 -s ho-0 --eval`）：真值 `EVT-1 台风封港@WH-003 frame46→SHP-0007,SHP-0011`；
`EVT-2 道路中断@WH-001 frame47→SHP-0023`；答案 `{NEWS-001:[SHP-0007,SHP-0011], NEWS-002:[SHP-0023]}`；
污染：别名5/单位漂移5/状态置空5/编码乱码1；判别力 `naive=0.000 linked=0.333 oracle=1.000 gap=+0.333`。

## 6. 覆盖与未覆盖

**已覆盖（29 用例分布）：**

- 确定性 / 逐字节可复现（含 seed 生效、旋钮夹紧、未知域返回 `None`）。
- 真值先建、恒可恢复（oracle 满分、hub 互异、答案按观测 id 编键、真值不泄漏、真值内部自洽）。
- 脏度只动观测、可逆、随 d 缩放（6 类污染中的 unit/garble 逐条对账；零脏度绝对干净；真值不被碰）。
- 判别区间与鲁棒性曲线（naive 满分/崩溃两端、linked 相对更优、均值单调退化、oracle 恒 1）。
- 物化四形态正确（JSON round-trip、CSV 行数、SQLite 真实可查询、preview 确定性文本）。
- CLI 三子命令接线（退出码、写文件、未知域非零退出）。

**未覆盖（如实列出的盲区）：**

- **性能 / 并发**：无基准、无耗时上限断言、无多线程/多进程测试（生成器为纯函数、无共享可变状态，
  但这一点未被测试固化）。
- **大规模 property-based / 模糊测试**：`test_solvers` 里的「跨 8 seed」是**手写的小样本**遍历，
  并非 Hypothesis 之类的属性测试；未在大 seed 空间或大 `params`（更多 warehouses/shipments/frames）
  上做属性验证。
- **真实数据校准的校验**：无。数理分布是**手设合成示意，未逆向自真实数据**（no real-data calibration），
  因此也**不存在**、且不可能有针对「贴合真实分布」的测试——这是设计边界，非遗漏。
- **`linked` 的 L2→L5 细粒度**：未测（也测不出），因为确定性 `linked` 本就只区分 L1 vs L≥2，
  L2–L5 同分 ~0.8。真正的语义梯度留给未来 LLM/axiom 解法。
- **多模态与 LLM 基准**：**无** PDF/NoSQL 多模态、**无** LLM 基准跑——那是母项目 Prism 的另一条线，
  不在本包，故本包测试也不涉及。
- **其它污染类别的逐条对账**：`test_dirtiness` 对 unit-drift、garbled 做了逐条对账，但对
  status-null / time-offset / throughput-freeze 只测了「随 d 缩放 / 计入 corruption_map」层级，
  未逐条对账其可逆性。
- **`anomaly_cause` 任务**：solver 与打分逻辑支持 `explain_delays` 与 `anomaly_cause` 两个任务，
  但判别力测试只覆盖 `explain_delays`（`naive_solve`/`linked_solve` 对非 `explain_delays` 直接返回空）。

## 优缺点（pros & cons）

### 优点

- **围绕不变量而非函数组织**：6 个文件对齐 6 条支柱不变量（确定性 / 真值可恢复 / 脏度只动观测 /
  判别区间 / 物化正确 / CLI 接线），断言直接是生成器的对外契约，可当可执行文档读。
- **确定性使测试廉价且稳定**：无 `random`、无时钟，`test_same_knobs_byte_identical` 直接逐字节比对整个包；
  全套 29 用例约 **0.28s** 完成，无网络/无外部进程，适合做提交前门禁，且不会有 flaky。
- **旋钮×真值的关键不变量被网格化验证**：`test_truth_invariant_to_dirtiness_and_link` 在
  4×5=20 格 `(dirtiness, link)` 网格上确认真值答案恒定——这是本项目最重要承诺的高覆盖验证。
- **可逆污染有逐条对账**：unit-drift 与 garbled 做了「corruption_map 与实际观测**逐条一致**」的双向断言，
  而不只是「大概变脏了」。
- **SQLite 是真查询而非存在性检查**：`test_sqlite_is_queryable` 跑真实 `SELECT ... WHERE status='delayed'`
  并与包内计数对齐，证明关系形态确实可用。
- **诚实纪律被固化进测试**：见第 4 节——把过强的「逐 seed 单调」弱化为真实成立的「均值单调」，
  并在 docstring 写明原因，而非挑 seed 凑绿。这类自我约束在测试层面能直接看到。

### 缺点 / 局限

- **判别力测试无法证明「真正的语义区分」**：确定性 `linked` 只区分 **L1 vs L≥2**，L2–L5 同分约 **0.8**
  （满分上限也是 ~0.8 而非 1.0）。因此 `test_solvers` 断言的是**结构性关系**（naive 崩、linked 更高、
  均值单调），**并不能**证明系统能做 L4/L5 的语义推理——那需要未来的 LLM/axiom 解法，本包测不到。
- **鲁棒性只在均值层面成立，单 seed 非单调**：第 4 节的反例（seed `ho-1`：脏 0.9 的 0.727 > 脏 0.0 的 0.714）
  是真实的；测试只能断言 8-seed 均值单调（0.667→0.627→0.573→0.554），**不能**断言任意单 seed 单调。
  读者若只看单条 seed 的 sweep 曲线，可能被台阶/回弹误导。
- **样本规模小、非属性测试**：「跨 8 seed」是硬编码小样本，非 Hypothesis 式属性测试；对更大参数规模、
  更大 seed 空间的行为无覆盖，边角失效可能漏网。
- **无性能 / 并发 / 大规模测试**：无耗时上限、无并发安全断言（尽管生成器是纯函数）。
- **无真实校准校验**（且不可能有）：数理分布是**手设合成示意，未逆向自真实数据**；「分布贴合现实」
  这件事既没测，也超出本包范围——它本就不声称做到。
- **部分污染类别只测到「缩放/入册」层级**：status-null / time-offset / throughput-freeze 未做
  unit-drift 那样的逐条可逆对账。
- **任务覆盖不全**：`anomaly_cause` 任务有代码路径与打分支持，但判别力测试只覆盖 `explain_delays`。
- **范围边界**：本包不含 PDF/NoSQL 多模态与 LLM 基准跑（属母项目 Prism 另一条线），故这些能力
  在本测试套件中**完全不存在**——不要据本套件推断这些能力已被验证。
